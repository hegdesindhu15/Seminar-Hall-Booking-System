# Seminar Hall Booking Management System

A web-based application developed to automate the process of seminar hall booking in an academic institution.  
The system replaces manual booking methods with a centralized solution that prevents scheduling conflicts and improves resource utilization.

---

## Problem Statement

Managing seminar hall bookings manually through phone calls, emails, or in-person requests leads to inefficiencies, errors, and scheduling conflicts.  
There is a need for an automated system that provides transparency, accuracy, and ease of access.

---

## Solution Overview

This system provides:
- Centralized booking management
- Real-time availability checking
- Conflict-free scheduling
- Easy update and cancellation of bookings

---

## Features

- User registration and login  
- View list of seminar halls  
- Book seminar halls for specific dates and time slots  
- Conflict detection to avoid overlapping bookings  
- Update booking details  
- Cancel bookings  
- View scheduled and booked halls  

---

## Tech Stack

- **Frontend:** HTML  
- **Backend:** PHP  
- **Database:** MySQL  
- **Server:** XAMPP (Apache + MySQL)  

---

## Modules Included

- Register / Login  
- Seminar Halls List  
- Book Hall  
- Show Schedules  
- Update Booking  
- Cancel Booking  
- Logout  

---

## How the System Works

- User details and booking information are stored in a MySQL database.
- When a booking request is made, the system checks existing schedules to avoid conflicts.
- All booking operations (create, read, update, delete) are handled using PHP.
- XAMPP provides the local development and testing environment.
